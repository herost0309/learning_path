# 架构对比图：应用是否应使用 DB 存储 Metadata

---

## 图一：当前争议的核心矛盾

```
┌─────────────────────────────────────────────────────────────────┐
│                     架构师的观点                                  │
│                                                                   │
│   "应用 = 通道 → 不需要DB → metadata 放上下游"                    │
│                                                                   │
│   ┌──────────┐     ┌──────────────────────┐     ┌──────────┐     │
│   │  上游系统  │────▶│  本应用（无状态通道）  │────▶│  下游系统  │     │
│   │  存metadata│     │  只做转发，不存储      │     │  存metadata│     │
│   └──────────┘     └──────────────────────┘     └──────────┘     │
│                                                                   │
│   问题：流程走到哪一步？谁在审批？→ 上下游都不知道！                │
└─────────────────────────────────────────────────────────────────┘
```

## 图三：如果按架构师方案（无 DB）会怎样？

```
    ┌──────────┐         ┌─────────────────────────────────┐
    │  上游系统A │         │         本应用（无DB）            │
    │           │───API──▶│                                 │
    │ metadata  │  ?慢?   │  问题1: 文档怎么检索？            │
    │ 在这里    │  ?挂?   │        → 只能每次调上游API查      │
    │           │  ?变?   │        → 上游挂了 = 平台瘫痪     │
    └──────────┘         │                                 │
                         │  问题2: 流程状态存哪？            │
    ┌──────────┐         │        → 上下游都不知道流程状态   │
    │  上游系统C │───API──▶│        → 工作流引擎无法运转 ❌    │
    │           │  ?无API? │                                 │
    │ metadata  │         │  问题3: 配置和权限存哪？          │
    │ 在这里    │         │        → 平台配置不能放业务系统   │
    └──────────┘         │                                 │
                         │  问题4: 新区域部署？              │
    ┌──────────┐         │        → 必须等所有上下游就绪     │
    │  下游系统B │◀──API──│        → 部署周期: 天 → 周/月    │
    │           │         │                                 │
    │ metadata  │         └─────────────────────────────────┘
    │ 在这里    │
    └──────────┘

    ⚠️ 结论：无DB方案下，4类metadata中有3类无处可存，平台核心功能无法实现
```

## 图四：数据主权边界（最关键的一张图）

```
    ┌───────────────────────────────────────────────────────────┐
    │                     数据主权边界                           │
    │                                                           │
    │    上下游拥有                    本平台拥有                │
    │    ──────────                    ──────────                │
    │                                                           │
    │    • 业务订单数据                • 文档索引/目录            │
    │    • 客户信息                    • 流程实例状态             │
    │    • 合同条款                    • 任务分配/审批记录        │
    │    • 产品目录                    • 配置/映射规则            │
    │    • 财务数据                    • 租户/权限信息            │
    │    • ...                        • ...                     │
    │                                                           │
    │    架构师说的"通道"精神 ──▶    这些是我们必须自存的 ──▶     │
    │    业务数据不属于我们            它们是平台运行的必需品      │
    │                                                           │
    │    ✅ 我们同意这部分               ❌ 这部分不能没有DB       │
    └───────────────────────────────────────────────────────────┘
```

## 图五：多区域部署对比

```
    【方案A: 有DB — 标准化部署包】              【方案B: 无DB — 依赖链部署】

    ┌─────────────────────┐                  ┌─────────────────────┐
    │  区域1 (us-east-1)   │                  │  区域1 (us-east-1)   │
    │                     │                  │                     │
    │  App + PG + S3      │                  │  App（无DB）         │
    │  ✅ 自包含，一键部署  │                  │  ❌ 等上游A/B/C就绪   │
    └─────────────────────┘                  └─────────────────────┘

    ┌─────────────────────┐                  ┌─────────────────────┐
    │  区域2 (eu-west-1)   │                  │  区域2 (eu-west-1)   │
    │                     │                  │                     │
    │  App + PG + S3      │                  │  App（无DB）         │
    │  ✅ 同样一键部署     │                  │  ❌ 等另一套上游就绪  │
    └─────────────────────┘                  │     (可能不同供应商)  │
                                             └─────────────────────┘
    部署周期: 1-2天                           部署周期: 2-4周
    区域扩展: 线性成本                         区域扩展: 非线性依赖
```

    ┌───────────────────────────────────────────────────────────┐
    │                    通用平台（有 DB）                        │

    │                                                           │

    └───────────────────────────────────────────────────────────┘
