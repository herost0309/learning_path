# Skill: AI 辅助 Code Review

> 用途：标准化代码审查流程，结合 AI 自动检测 + 人工审查。

## 触发条件

- 收到 Merge Request / Pull Request
- 代码审查环节

## 审查维度

### 维度 1: 功能正确性

- [ ] 代码是否实现了 Spec 描述的功能
- [ ] 边界情况是否处理
- [ ] 错误处理是否完整
- [ ] 是否有明显的逻辑错误

### 维度 2: 安全性

- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 风险
- [ ] 是否有敏感信息泄露（日志、响应）
- [ ] 权限校验是否完整
- [ ] 是否有命令注入风险

### 维度 3: 性能

- [ ] 是否有 N+1 查询
- [ ] 数据库查询是否有合适索引
- [ ] 是否有不必要的全量查询
- [ ] 是否有内存泄漏风险（大集合、未关闭资源）

### 维度 4: 可维护性

- [ ] 命名是否清晰、一致
- [ ] 函数/方法是否过长（> 50 行需关注）
- [ ] 是否有重复代码
- [ ] 注释是否充分（特别是复杂逻辑处）

### 维度 5: 测试

- [ ] 是否有对应的测试
- [ ] 测试是否覆盖核心场景
- [ ] 测试是否可靠（不依赖外部状态）

### 维度 6: 规范符合度

- [ ] 是否遵循 CLAUDE.md 中的编码规范
- [ ] 是否遵循项目架构分层
- [ ] 提交信息格式是否正确

## 执行步骤

### Step 1: AI 自动扫描

AI Agent 执行以下自动检查：

1. **静态分析**: 运行 lint 工具
2. **安全扫描**: 检查常见漏洞模式
3. **复杂度分析**: 标记高复杂度函数
4. **规范检查**: 对照 CLAUDE.md 规范
5. **测试覆盖**: 检查变更文件的测试覆盖情况

生成 **AI Review Report**：
```
## AI Code Review Report

### 概览
- 文件数: [N]
- 变更行数: +[N] / -[N]
- 风险等级: [低/中/高]

### 发现的问题
| # | 类型 | 严重度 | 文件:行号 | 描述 |
|---|------|--------|-----------|------|
| 1 | 安全 | 高 | foo.js:42 | 潜在SQL注入 |

### 建议
- [具体建议]
```

### Step 2: 人工深度审查

开发者根据 AI 报告，重点审查：

1. **高风险项**: AI 标记的高严重度问题
2. **业务逻辑**: AI 无法判断的业务正确性
3. **架构影响**: 改动对整体架构的影响
4. **AI 生成代码**: 如果标记了 `[AI-Generated]`，额外关注架构合理性

### Step 3: 反馈与修改

- 审查意见分为：`Must Fix` / `Suggestion` / `Question`
- `Must Fix` 项必须修改后才能合并
- 提交修改后触发重新审查

## 注意事项

- AI Review 是辅助，不替代人工审查
- 安全和业务逻辑相关变更必须人工审查
- 保持 Review 及时响应，避免成为瓶颈
- AI 标记的误报可以记录，优化后续规则
