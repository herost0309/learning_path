```

├── README.md              # 本文件

│   ├── code-generation.md # 代码生成 Prompt

```

## 贡献指南

### 案例文档命名

`case-studies/[YYYY-MM-DD]-[简短描述].md`

例如：`case-studies/2026-05-15-legacy-order-module-test-generation.md`

### 案例文档格式

```markdown
# [案例标题]

## 背景
[场景描述]

## 使用方法
[具体做法：Prompt、Skill、工具配置]

## 效果
- 量化数据: [具体数字]
- 定性感受: [体验]

## 经验教训
[总结]

## 适用场景
[推荐使用场景]
```

## 精选 Prompt 技巧

### 代码生成

```
角色：你是一个 [语言/框架] 资深开发者。
任务：根据以下 Spec 实现 [功能名称]。
约束：
- 遵循项目编码规范（见 CLAUDE.md）
- 使用项目已有的 [框架/库]
- 包含错误处理和参数校验
- 包含单元测试

Spec:
[粘贴 Spec 内容]
```

### 调试辅助

```
角色：你是一个调试专家。
问题描述：[现象]
错误日志：
[粘贴日志]
相关代码：
[粘贴代码]

请分析可能的根因，并给出排查步骤。
```

### 重构辅助

```
角色：你是一个代码重构专家。
目标：重构以下代码，提高可读性和可维护性。
约束：
- 不改变外部行为
- 遵循 [SOLID/DRY/KISS] 原则
- 每步改动要小且可验证

代码：
[粘贴代码]
```
